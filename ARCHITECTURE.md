# Teresa Homeschool 架构说明

## 正确的系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户浏览器                                │
│                    (访问 teresa-homeschool.top)                │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    腾讯云服务器 (静态服务)                        │
│                  只做静态文件托管和反向代理                       │
│    不直接调用任何AI API，只转发请求到龙虾机                       │
└─────────────────────────────┬───────────────────────────────────┘
                              │
            (Tailscale内网 / 公网转发)
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                     龙虾机 (本地 Mac mini)                       │
│  ┌─────────────────┐  ┌──────────────────┐  ┌───────────────┐ │
│  │  前端静态服务   │  │   API服务        │  │ 数学内容生成器 │ │
│  │  (8848端口)    │  │   (3000端口)    │  │ (定时任务)     │ │
│  └─────────────────┘  └────────┬─────────┘  └───────┬───────┘ │
│                                │                     │         │
│                         ┌──────▼──────┐              │         │
│                         │  豆包API    │              │         │
│                         │  讯飞星火   │◄─────────────┘         │
│                         │  (AI云服务) │                        │
│                         └─────────────┘                        │
└─────────────────────────────────────────────────────────────────┘
```

## 端口分配

| 端口 | 服务 | 说明 |
|------|------|------|
| 8848 | 前端静态 | 网站界面 |
| 3000 | API服务 | AI对话接口 |
| 8888 | 网站入口 | 腾讯云nginx端口 |

## 腾讯云服务器职责
- 静态托管 index.html
- 反向代理 `/api/*` 请求到龙虾机
- **不调用任何AI API**

## 龙虾机职责
- 运行 API 服务 (api-server.js)
- 调用豆包/讯飞星火AI
- 生成数学内容 (teresa-math-generator)
- 每天6点前生成当天数学课件和练习

## 需要修改的文件
1. server/index.js - 移除所有AI API调用，改为代理到本地
2. nginx配置 - 反向代理到龙虾机
3. 前端 - 确保调用本地API
